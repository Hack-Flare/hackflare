## Quick Commands

**Development & Building:**
- `npm run dev` — Start dev server (localhost:5173)
- `npm run build` — Production build output to `./build`
- `npm start` — Run production server from build
- `npm run typecheck` — TypeScript type checking + route generation
- `npm run format` — Format code with Prettier

## Architecture

**React Router v7 with File-Based Routing**
- Routes defined in `app/routes/` directory, with `routes.ts` as route config
- Entry point: `app/root.tsx` (Layout + ErrorBoundary + Outlet)
- Uses React Router's newer data-driven approach; routes are auto-generated by React Router

**UI Components (shadcn/ui)**
- Components live in `app/components/ui/` — added via `npx shadcn@latest add <component>`
- All imports use path alias: `~/components/ui/button` (not relative paths)
- Built on Radix UI primitives with Tailwind CSS styling

**Styling**
- Tailwind CSS v4 (via `@tailwindcss/vite`)
- CSS file: `app/app.css`
- Prettier plugin (`prettier-plugin-tailwindcss`) auto-sorts classes
- Class utilities: `cn()` and `cva()` functions in `~/lib/utils.ts`

**TypeScript Setup**
- Strict mode enabled
- Path alias `~/*` → `./app/*` (configured in `tsconfig.json`)
- Server-side rendering (SSR) enabled by default in `react-router.config.ts`

## Key Files & Conventions

- **`app/root.tsx`** — Layout component + error boundary (required)
- **`app/routes.ts`** — Route config (file-based routes map)
- **`app/lib/utils.ts`** — `cn()` utility for Tailwind class merging
- **`app/app.css`** — Global styles + Tailwind directives
- **`.prettierrc`** — Prettier config with tailwindcss plugin (class sorting, printWidth: 80)

## Adding Components

```bash
npx shadcn@latest add button
```

Components go to `app/components/ui/` automatically. Import with `~/components/ui/button`.
